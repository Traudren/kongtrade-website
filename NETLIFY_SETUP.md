# Инструкция: Добавление NETLIFY_AUTH_TOKEN для исправления ошибки деплоя

## Шаг 1: Создание Personal Access Token

1. Откройте [Netlify Dashboard](https://app.netlify.com/)
2. Нажмите на иконку профиля в правом верхнем углу
3. Выберите **"User settings"** (Настройки пользователя)
4. В левом меню выберите **"Applications"**
5. Найдите раздел **"Personal access tokens"**
6. Нажмите **"New access token"** (Создать новый токен)
7. Введите название токена (например: "KongTrade Build Token")
8. Нажмите **"Generate token"** (Создать токен)
9. **ВАЖНО:** Скопируйте токен сразу - он показывается только один раз!
   - Токен будет выглядеть примерно так: `nfp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

## Шаг 2: Добавление токена в Environment Variables

1. Вернитесь в **Netlify Dashboard**
2. Выберите ваш сайт (KongTrade)
3. Перейдите в **"Site settings"** (Настройки сайта)
4. В левом меню выберите **"Build & deploy"**
5. Прокрутите вниз до раздела **"Environment"**
6. Нажмите **"Environment variables"**
7. Нажмите **"Add a variable"** (Добавить переменную)
8. Заполните форму:
   - **Key:** `NETLIFY_AUTH_TOKEN`
   - **Value:** (вставьте скопированный токен из Шага 1)
   - **Scopes:** оставьте как есть (Build time)
9. Нажмите **"Save"** (Сохранить)

## Шаг 3: Запуск нового деплоя

1. В Netlify Dashboard выберите ваш сайт
2. Перейдите в раздел **"Deploys"** (Деплои)
3. Нажмите **"Trigger deploy"** → **"Clear cache and deploy site"**
4. Дождитесь завершения деплоя

## Проверка

После деплоя проверьте:
- ✅ Деплой должен завершиться успешно без ошибок плагина
- ✅ Сайт должен работать корректно
- ✅ API routes должны функционировать

## Если токен потерян

Если вы потеряли токен, просто создайте новый (Шаг 1) и обновите переменную окружения (Шаг 2).

## Безопасность

⚠️ **Важно:** Никогда не публикуйте токен в публичных репозиториях или не делитесь им публично. Токен дает доступ к вашему Netlify аккаунту.
